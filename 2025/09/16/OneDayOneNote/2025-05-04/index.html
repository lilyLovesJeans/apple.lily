2025-05-04-建立 MCP Node 範例專案指令說明

  1. 建立 MCP Node 範例專案（從零開始）

在家目錄建立資料夾並安裝 SDK：

mkdir ~/mcp-server
cd ~/mcp-server
npm init -y
npm i @modelcontextprotocol/sdk zod

請解釋這些指令

好，我幫你逐行拆解這些指令，說明它們的作用，以及為什麼要這樣做。

1️⃣ 建立專案資料夾

  • mkdir = make directory，建立資料夾的指令。

  • ~/mcp-server

    • ~ 代表「使用者的家目錄」（macOS 預設就是 /Users/你的帳號）。
    • mcp-server 是資料夾名稱。
      👉 這一步會在你的「家目錄」下新增一個叫做 mcp-server 的資料夾，用來存放 MCP 專案。

2️⃣ 進入專案資料夾

  • cd = change directory，切換到剛剛建立的 mcp-server 資料夾裡。
    👉 接下來的所有指令都會在這個資料夾底下執行。

3️⃣ 初始化 Node.js 專案

  • npm = Node Package Manager，Node.js 的套件管理工具。

  • init = 初始化一個新的專案，會建立 package.json 檔案，這個檔案用來紀錄：

    • 專案名稱、版本
    • 專案的相依套件（dependencies）
    • 執行指令（scripts）

  • -y = yes，自動接受所有預設選項，不需要手動輸入。
    👉 這一步會產生一個 package.json，作為你的 Node.js 專案的「配置檔」。

4️⃣ 安裝 SDK 與 Zod

  • npm i = npm install，安裝套件。

  • @modelcontextprotocol/sdk

    • MCP（Model Context Protocol）的官方 JavaScript/TypeScript SDK。
    • 這個套件提供 MCP server 開發需要的 API 與工具。

  • zod

    • 一個 TypeScript/JavaScript 的 資料驗證庫。
    • 常用來檢查輸入的 JSON 或物件是否符合預期的格式。
      👉 安裝後會更新 package.json，並且把套件下載到 node_modules 資料夾。

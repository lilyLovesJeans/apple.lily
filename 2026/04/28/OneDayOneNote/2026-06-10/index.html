GPT:
my.cnf 是 MySQL 的主要配置文件，用於設置 MySQL 伺服器的各種參數，比如端口號、數據目錄、緩存大小、網絡設置等。以下是一些常見的配置選項以及如何配置它們的建議。

my.cnf 的位置

my.cnf 文件通常可以在以下幾個位置找到，具體位置取決於 MySQL 的安裝方式：

  • /etc/my.cnf 或 /etc/mysql/my.cnf（全局配置文件）
  • /usr/local/mysql/etc/my.cnf（自定義安裝的 MySQL）
  • ~/.my.cnf（用戶級別的配置）

[client] 部分

這一部分配置 MySQL 客戶端的參數，例如命令行工具使用的設置：

  • **port**：指定默認的 MySQL 端口號（通常為 3306）。
  • **socket**：指定 Unix Socket 文件的位置，通常用於本地連接。

[mysqld] 部分

這是 MySQL 伺服器的主要配置部分，控制伺服器的各種行為：

  1. 基本設置

    • **user**：指定運行 MySQL 伺服器的用戶（默認為 mysql）。
    • **port**：指定 MySQL 使用的端口號（默認為 3306）。
    • **bind-address**：指定 MySQL 綁定的 IP 地址。通常設為 127.0.0.1 以僅允許本地連接。如果想允許遠程連接，可以設置為 0.0.0.0 或特定 IP。
    • **datadir**：指定 MySQL 數據存儲目錄的位置。

  2. 性能調整

    • **max_connections**：指定 MySQL 允許的最大連接數，默認為 151。如果是大流量應用，應根據硬體資源增加此值。
    • **skip-external-locking**：防止 MySQL 對文件鎖使用外部鎖定，這樣可以提高性能。

  3. 語言與字符集設置

    • **character-set-server**：設置伺服器的默認字符集，通常設為 utf8mb4，以支援更多特殊字符。
    • **collation-server**：設置伺服器的字符集校對規則。

  4. 緩存與內存設置

    • **key_buffer_size**：設置索引緩存大小，適合 MyISAM 表使用。InnoDB 則主要依賴 innodb_buffer_pool_size。
    • **query_cache_size**：查詢緩存大小，可以加快相同查詢的響應速度。請注意，從 MySQL 8.0 開始已棄用查詢緩存。
    • **innodb_buffer_pool_size**：設置 InnoDB 的緩存大小，建議設置為系統內存的 50%-80%，以提高性能。

  5. 日誌設置

    • **log_error**：指定錯誤日誌文件的路徑，用於記錄 MySQL 伺服器運行時發生的錯誤。
    • **slow_query_log**：是否啟用慢查詢日誌（1 為啟用）。
    • **slow_query_log_file**：指定慢查詢日誌文件的路徑。
    • **long_query_time**：設置查詢的時間閾值（單位為秒），超過這個時間的查詢會被記錄在慢查詢日誌中。

  6. InnoDB 引擎設置

    • **innodb_log_file_size**：設置 InnoDB 日誌文件的大小，通常設置為緩存池大小的 1/4。
    • **innodb_file_per_table**：為每個表創建單獨的數據文件，便於管理和備份。

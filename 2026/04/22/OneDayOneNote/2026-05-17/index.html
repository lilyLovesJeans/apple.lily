問題
問題： 為什麼 OOP 中需要封裝數據？
通過訪問修飾符（例如 public、private）來控制屬性和方法的可見性，保護數據並隱藏實現細節。
例如:
public class BankAccount
{
private decimal balance;

public void Deposit(decimal amount)
{
    if (amount > 0)
    {
        balance += amount;
        Console.WriteLine($"存入 {amount} 元，目前餘額為 {balance} 元");
    }
}

public decimal GetBalance()
{
    return balance;
}

}

問題： 繼承如何幫助代碼的重用？
允許子類別從父類別繼承屬性和方法，從而實現代碼的重用。
例如:
public class Animal
{
public void Eat()
{
Console.WriteLine(“Eating…”);
}
}

public class Dog : Animal
{
public void Bark()
{
Console.WriteLine(“Barking…”);
}
}

class Program
{
static void Main(string[] args)
{
Dog myDog = new Dog();
myDog.Eat(); // 繼承自父類別 Animal
myDog.Bark(); // 自己的方法
}
}

修正後的回答

問題 1：為什麼 OOP 中需要封裝數據？

回答：
封裝數據的主要目的是保護對象的內部狀態，防止外部程式直接訪問和修改內部的數據，從而維持系統的穩定性和安全性。通過使用訪問修飾符（例如 public 和 private），我們可以控制屬性和方法的可見性，並確保只有經過驗證或特定的方式才能修改數據。這樣做也有助於隱藏實現細節，使得類別對使用者來說更易於理解和使用。

範例：

在這個範例中，balance 是私有的，只有通過 Deposit 方法和 GetBalance 方法才能存取和操作餘額。

問題 2：繼承如何幫助代碼的重用？

回答：
繼承允許子類別從父類別繼承屬性和方法，這樣可以實現代碼的重用，減少重複編寫相同功能的代碼。父類別中定義的所有屬性和方法都可以被子類別直接使用，而子類別也可以添加自己的特性或方法，或者覆寫父類別的方法來實現特殊化的行為。這種方式不僅減少了代碼重複，也使得程式的結構更有層次，更易於擴展。

在這個例子中，Dog 類別繼承了 Animal 類別，因此可以直接使用 Animal 類別的 Eat 方法，而不需要再次定義，從而實現了代碼重用。

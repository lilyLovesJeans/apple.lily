類別與實例的基本概念

在 C# 中，類別（Class） 是一個藍圖，它定義了物件的屬性和方法。而實例（Instance） 是根據這個類別創建的具體物件。簡單來說，類別像是一個模具，而實例就是用模具做出來的具體物件。

非靜態方法的呼叫

假設你有一個 Person 類別，定義了一些屬性和方法。

範例：

public class Person
{
    public string Name;

    public void Speak()
    {
        Console.WriteLine($"Hello, I am {Name}");
    }
}

在這個 Person 類別中，Speak 是一個非靜態方法。這意味著如果你想要呼叫 Speak 方法，你必須先創建一個 Person 類別的實例，也就是「創建物件」。

如何創建實例：

class Program
{
    static void Main(string[] args)
    {
        // 創建一個 Person 的實例
        Person person1 = new Person();
        person1.Name = "Alice";
        person1.Speak(); // 輸出: Hello, I am Alice
    }
}

在這裡，我們使用 new Person() 來創建一個名為 person1 的物件（實例），然後才能呼叫 Speak 方法。如果沒有創建 person1 這個物件，是無法直接呼叫 Speak 的。

靜態方法的呼叫

現在我們來看什麼是靜態方法，以及它如何在不創建類別實例的情況下被呼叫。

範例：靜態方法

public class MathUtilities
{
    public static void PrintPi()
    {
        Console.WriteLine("Pi is approximately 3.14159");
    }
}

在這個範例中，PrintPi 是一個靜態方法，定義在 MathUtilities 類別中。注意它前面有一個關鍵字 static，這表示這個方法屬於類別本身，而不是某個特定的物件。

如何呼叫靜態方法：

class Program
{
    static void Main(string[] args)
    {
        // 直接通過類別名稱呼叫靜態方法
        MathUtilities.PrintPi(); // 輸出: Pi is approximately 3.14159
    }
}

在這裡，你可以看到我們並沒有創建 MathUtilities 的物件，而是直接使用類別名稱來呼叫 PrintPi 方法。這就是「在不創建類別實例的情況下被呼叫」。這樣做的原因是靜態方法的邏輯通常是和類別整體相關的，而不是和某個特定的物件相關。

具體情境舉例

讓我們來看一個更具體的情境來說明靜態方法為什麼可以不需要創建物件。

情境 1：計算工具

假設你要創建一個計算工具的類別，提供一些常用的數學方法，例如計算圓的面積。這些方法只需要根據輸入的數據進行計算，而不需要記住任何特定物件的狀態。

範例：

public class MathHelper
{
    public static double CalculateCircleArea(double radius)
    {
        return Math.PI * radius * radius;
    }
}

在這裡，CalculateCircleArea 方法是靜態的，因為它的功能是根據輸入的半徑計算圓的面積，與物件的狀態無關。

如何使用這個靜態方法：

class Program
{
    static void Main(string[] args)
    {
        double area = MathHelper.CalculateCircleArea(5);
        Console.WriteLine($"圓的面積是: {area}");
    }
}

在這個例子中，我們不需要創建 MathHelper 的實例，而是直接呼叫類別中的靜態方法來進行計算。

情境 2：應用程式的入口點

Main 方法 就是另一個例子。當應用程式啟動時，C# 需要知道從哪裡開始執行。由於 Main 方法是程式的入口點，而且不需要物件的狀態來運行，因此它必須是靜態的。這樣，C# 執行環境就可以直接呼叫它來啟動程式，而不需要創建任何物件。

範例：

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("這是程式的入口點！");
    }
}

這個 Main 方法是靜態的，這樣 C# 就可以在啟動應用程式時直接呼叫它，而不需要先創建 Program 類別的實例。

小結

  • 靜態方法（static）：屬於類別本身，不需要創建物件就能呼叫。適合那些和特定物件無關的功能，如數學計算、應用程式入口點等。
  • 非靜態方法：屬於類別的某個實例，需要先創建物件才能呼叫。適合那些需要物件的特定狀態（例如物件的屬性值）的操作。

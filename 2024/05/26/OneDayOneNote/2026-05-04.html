第一題：類別和物件之間的關係

問題 1
Person 是一個類別，而 person1 是一個物件。請描述它們之間的關係。

答案：
Person 是 person1 的模板（藍圖），person1 是根據 Person 這個模板所創建的具體實例（物件），它可以擁有具體的屬性。

範例：

定義 Person 類別：

public class Person
{
    public string name;
    public string sex;

    public void DisplayInfo()
    {
        Console.WriteLine($"My name is {name}, sex is {sex}.");
    }
}

創建 person1 物件：

class Program
{
    static void Main(string[] args)
    {
        Person person1 = new Person();
        person1.name = "Bobby";
        person1.sex = "Male";
        person1.DisplayInfo(); // 輸出: My name is Bobby, sex is Male.
    }
}

第二題：定義車子的類別

問題 2
如果你想創建多個擁有不同屬性的「車」物件，你應該先定義什麼？

答案：
我要先定義這些物件的藍圖，也就是類別。

範例：

定義 Car 類別：

public class Car
{
    public string color;
    public string brand;
    public string style;

    public void DisplayCarInfo()
    {
        Console.WriteLine($"This car is a {color} {brand}, and its style is {style}.");
    }
}

創建 Car 物件：

class Program
{
    static void Main(string[] args)
    {
        Car car1 = new Car();
        car1.color = "Red";
        car1.brand = "Toyota";
        car1.style = "SUV";
        car1.DisplayCarInfo(); // 輸出: This car is a Red Toyota, and its style is SUV.

        Car car2 = new Car();
        car2.color = "Blue";
        car2.brand = "Honda";
        car2.style = "Sedan";
        car2.DisplayCarInfo(); // 輸出: This car is a Blue Honda, and its style is Sedan.
    }
}

頂層語句的問題回答

問題 1：為什麼在頂層語句中不需要顯式地定義 Main 方法？

答案：
在 頂層語句（Top-level Statements） 中，C# 提供了一種更簡潔的方式來撰寫程式碼。這樣做的目的是讓開發者可以直接撰寫可執行的代碼，編譯器會自動把這些頂層語句封裝到一個隱藏的 Main 方法中。因此開發者不用手動撰寫 Main 方法，這樣除錯時也能更專注於自己撰寫的核心邏輯。

問題 2：頂層語句有哪些類型的代碼可以被包括？

答案：

  1. 變數定義：

    int num1 = 11;

  2. 條件語句：

    if (num1 < 10)
{
    Console.WriteLine($"number: {num1} < 10");
}
else

}

  3. 循環語句：

}

  4. 方法呼叫：

2025-05-13 npm（Node Package Manager）的常見指令與參數

npm（Node Package Manager）的常見指令與參數，讓你知道什麼時候該怎麼下。

🟢 常用的 npm 指令

1. 建立專案

npm init
  • 建立 package.json 檔案
  • 會問一堆問題（專案名稱、版本、描述…）
npm init -y
  • 快速建立，用預設值，不會問問題

2. 安裝套件

npm install <套件名稱>

或縮寫：

npm i <套件名稱>

常見參數：

  • --save（舊寫法，現在預設就是存進 dependencies）

  • --save-dev 或 -D：安裝成開發用套件（只在開發時需要，正式部署不會用到）
    👉 例如 eslint、nodemon

    npm i -D nodemon

  • -g：全域安裝（整台電腦都能用，不限某專案）
    👉 例如：

    npm i -g typescript

  • --force：強制安裝（忽略版本衝突或警告）

  • --legacy-peer-deps：遇到新版相依性問題時，用舊方式安裝

3. 移除套件

npm uninstall <套件名稱>

或縮寫：

npm un <套件名稱>

4. 更新套件

npm update <套件名稱>

更新所有套件：

npm update

5. 執行指令

在 package.json 裡的 "scripts" 會設定指令，例如：

"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js"
}

執行：

npm run start
npm run dev

如果是特殊的 start，可以直接用：

npm start

6. 查看套件版本

npm list
  • 顯示目前專案安裝的套件
npm list <套件名稱>
  • 只看某個套件版本
npm -v
  • 看 npm 版本
node -v
  • 看 Node.js 版本

7. 清理

npm cache clean --force
  • 清除快取（有時候安裝壞掉會用這個）
rm -rf node_modules
npm install
  • 刪掉全部套件，重新安裝（解決奇怪錯誤的常見招數）

📌 常見用法總整理

  1. 建立專案：npm init -y
  2. 安裝套件：npm i express dotenv
  3. 安裝開發工具：npm i -D nodemon
  4. 移除套件：npm un express
  5. 執行 script：npm run dev

階段 1：類別與物件的基本概念
解釋
類別（Class） 是物件導向程式設計（OOP）的核心概念之一。類別是物件的藍圖，定義了物件的屬性（數據）和方法（行為）。物件（Object） 是類別的實例，擁有類別中定義的屬性和方法。
範例：
public class Person
{
// 屬性
public string Name;
public int Age;

// 方法
public void Speak()
{
    Console.WriteLine($"Hello, my name is {Name} and I am {Age} years old.");
}

}
在這個範例中，Person 是一個類別，具有 Name 和 Age 兩個屬性，以及一個 Speak 方法。你可以根據這個類別創建多個 Person 物件，每個物件都有自己的 Name 和 Age 值。

階段 2：定義類別與創建物件
解釋
在 C# 中，定義一個類別後，可以通過 new 關鍵字創建該類別的物件。每個物件都是類別的獨立實例，擁有自己的數據和行為。
範例：
class Program
{
static void Main(string[] args)
{
// 創建 Person 物件
Person person1 = new Person();
person1.Name = “Alice”;
person1.Age = 30;
person1.Speak(); // 輸出: Hello, my name is Alice and I am 30 years old.

    Person person2 = new Person();
    person2.Name = "Bob";
    person2.Age = 25;
    person2.Speak(); // 輸出: Hello, my name is Bob and I am 25 years old.
}

}
在這個範例中，我們創建了兩個 Person 物件，person1 和 person2，並為它們設置不同的 Name 和 Age，然後呼叫 Speak 方法顯示各自的信息。

階段 3：存取修飾符（Access Modifiers）
解釋
存取修飾符 用來控制類別及其成員的可見性和訪問範圍。C# 中常用的存取修飾符有：
• public：公開的，任何地方都可以訪問。 
• private：私有的，僅在類別內部可訪問。 
範例：
public class Person
{
// 公開欄位
public string Name;

// 私有欄位
private int age;

// 公開方法
public void SetAge(int value)
{
    if (value > 0)
        age = value;
}

public int GetAge()
{
    return age;
}

}
在這個範例中，Name 是公開的，可以在任何地方訪問；而 age 是私有的，只能通過公開的方法 SetAge 和 GetAge 來修改和獲取。
問題
問題：
請說明 public 和 private 存取修飾符的區別，以及它們在類別成員中的應用場景。

階段 4：屬性（Properties）
解釋
屬性（Property） 是類別成員的一種，用於封裝欄位，提供安全的讀取和寫入方式。屬性通常有 get 和 set 訪問器。
範例：
public class Person
{
private string name;
private int age;

// 公開屬性
public string Name
{
    get { return name; }
    set { name = value; }
}

// 只讀屬性
public int Age
{
    get { return age; }
}

// 建構函數
public Person(string name, int age)
{
    this.name = name;
    this.age = age;
}

public void Speak()
{
    Console.WriteLine($"Hello, my name is {Name} and I am {Age} years old.");
}

}
在這個範例中，Name 屬性提供了讀寫訪問，而 Age 屬性僅提供讀取訪問。這樣可以控制外部程式碼對類別成員的操作。
問題
問題：
什麼是屬性（Property）？請解釋 get 和 set 訪問器的作用。

階段 5：建構函數（Constructors）
解釋
建構函數（Constructor） 是類別的一種特殊方法，用於初始化物件的狀態。建構函數的名稱與類別名稱相同，沒有返回類型。
範例：
public class Person
{
public string Name { get; set; }
public int Age { get; private set; }

// 無參數建構函數
public Person()
{
    Name = "Unknown";
    Age = 0;
}

// 有參數建構函數
public Person(string name, int age)
{
    Name = name;
    Age = age;
}

public void Speak()
{
    Console.WriteLine($"Hello, my name is {Name} and I am {Age} years old.");
}

}
在這個範例中，Person 類別有兩個建構函數：一個無參數建構函數，初始化 Name 為 “Unknown” 和 Age 為 0；另一個有參數建構函數，允許在創建物件時設置 Name 和 Age。
問題
問題：
什麼是建構函數（Constructor）？它在類別中有什麼作用？

階段 6：靜態成員（Static Members）
解釋
靜態成員（Static Members） 屬於類別本身，而不是類別的實例。靜態成員使用 static 關鍵字定義，可以直接通過類別名稱訪問，而不需要創建物件。
範例：
public class MathUtilities
{
// 靜態欄位
public static double Pi = 3.141592653589793;

// 靜態方法
public static double Add(double a, double b)
{
    return a + b;
}

public static double Multiply(double a, double b)
{
    return a * b;
}

}

class Program
{
static void Main(string[] args)
{
// 訪問靜態成員，不需要實例化類別
Console.WriteLine($”Pi 的值是 {MathUtilities.Pi}”);
double sum = MathUtilities.Add(5.5, 3.2);
double product = MathUtilities.Multiply(4.0, 2.5);
Console.WriteLine($”5.5 + 3.2 = {sum}”);
Console.WriteLine($”4.0 * 2.5 = {product}”);
}
}
在這個範例中，MathUtilities 類別包含靜態欄位 Pi 和靜態方法 Add、Multiply。這些成員可以直接通過類別名稱訪問，而不需要創建 MathUtilities 的物件實例。

階段 7：繼承（Inheritance）
解釋
繼承（Inheritance） 允許一個類別（子類別）從另一個類別（基類）繼承屬性和方法。這有助於重用代碼並建立類別之間的層次結構。
範例：
// 基類
public class Animal
{
public string Name { get; set; }

public void Eat()
{
    Console.WriteLine($"{Name} 正在吃食物。");
}

// 虛擬方法，允許派生類覆蓋
public virtual void Speak()
{
    Console.WriteLine($"{Name} 發出聲音。");
}

}

// 派生類
public class Dog : Animal
{
// 覆蓋基類的 Speak 方法
public override void Speak()
{
Console.WriteLine($”{Name} 汪汪叫。”);
}

// 派生類的特有方法
public void Fetch()
{
    Console.WriteLine($"{Name} 正在取回球。");
}

}

class Program
{
static void Main(string[] args)
{
Dog myDog = new Dog { Name = “Buddy” };
myDog.Eat(); // 輸出: Buddy 正在吃食物。
myDog.Speak(); // 輸出: Buddy 汪汪叫。
myDog.Fetch(); // 輸出: Buddy 正在取回球。

    // 基類引用指向派生類物件
    Animal myAnimal = myDog;
    myAnimal.Eat();   // 輸出: Buddy 正在吃食物。
    myAnimal.Speak(); // 輸出: Buddy 汪汪叫。
    // myAnimal.Fetch(); // 錯誤，基類引用無法訪問派生類特有方法
}

}
在這個範例中，Dog 類別繼承自 Animal 類別，並覆蓋了 Speak 方法。此外，Dog 類別還有一個特有的方法 Fetch。當使用基類引用 myAnimal 指向 Dog 物件時，呼叫 Speak 方法會執行 Dog 類別中的實現，但無法訪問 Fetch 方法。

問題：

階段 8：多態性（Polymorphism）
解釋
多態性（Polymorphism） 允許相同的接口在不同的類別中有不同的實現。這使得程式碼更加靈活和可擴展。
在 C# 中，多態性通常通過基類中的虛擬方法和子類中的覆蓋方法實現。
範例：
// 基類
public class Animal
{
public string Name { get; set; }

public virtual void Speak()
{
    Console.WriteLine($"{Name} 發出聲音。");
}

}

// 派生類
public class Cat : Animal
{
public override void Speak()
{
Console.WriteLine($”{Name} 喵喵叫。”);
}
}

public class Dog : Animal
{
public override void Speak()
{
Console.WriteLine($”{Name} 汪汪叫。”);
}
}

class Program
{
static void Main(string[] args)
{
List animals = new List
{
new Cat { Name = “Whiskers” },
new Dog { Name = “Rex” },
new Animal { Name = “Generic Animal” }
};

    foreach (var animal in animals)
    {
        animal.Speak();
    }

    // 輸出：
    // Whiskers 喵喵叫。
    // Rex 汪汪叫。
    // Generic Animal 發出聲音。
}

}
在這個範例中，Animal 類別定義了一個虛擬方法 Speak，Cat 和 Dog 類別覆蓋了這個方法。當使用基類引用 animal 呼叫 Speak 方法時，會根據實際物件的類型執行相應的覆蓋方法。

階段 9：封裝（Encapsulation）
解釋
封裝（Encapsulation） 是將數據（屬性）和操作數據的方法組織在一起，並控制對數據的訪問。這有助於保護數據不被外部不當修改，並確保物件的一致性。
範例：
public class BankAccount
{
// 私有欄位
private decimal balance;

// 公開屬性，僅提供讀取權限
public decimal Balance
{
    get { return balance; }
    private set { balance = value; }
}

// 建構函數
public BankAccount(decimal initialBalance)
{
    Balance = initialBalance;
}

// 公開方法，增加餘額
public void Deposit(decimal amount)
{
    if (amount > 0)
    {
        Balance += amount;
        Console.WriteLine($"已存入: {amount}，新餘額: {Balance}");
    }
}

// 公開方法，減少餘額
public void Withdraw(decimal amount)
{
    if (amount > 0 && amount <= Balance)
    {
        Balance -= amount;
        Console.WriteLine($"已提取: {amount}，新餘額: {Balance}");
    }
    else
    {
        Console.WriteLine("提取金額無效。");
    }
}

}

class Program
{
static void Main(string[] args)
{
BankAccount account = new BankAccount(1000m);
account.Deposit(500m); // 輸出: 已存入: 500， 新餘額: 1500
account.Withdraw(200m); // 輸出: 已提取: 200， 新餘額: 1300

    // 無法直接訪問私有欄位或屬性設置器
    // account.balance = 100; // 錯誤
    // account.Balance = 100; // 錯誤
}

}
在這個範例中，balance 欄位是私有的，只能通過公開的方法 Deposit 和 Withdraw 來修改。這確保了餘額的修改是受控制的，避免了外部程式碼的隨意更改。
